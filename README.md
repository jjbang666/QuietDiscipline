# 静心自律 (QuietDiscipline) 用户手册

## 📱 应用简介

**静心自律**是一款帮助您建立健康手机使用习惯的Android应用。通过智能时间管理、应用限制和正向激励，助您掌控数字生活，提升专注力与自律能力。

### 核心理念
- **自律不是惩罚，而是通往更好生活的路径**
- **柔性引导胜于强制限制**
- **数据驱动自我认知与改进**

---

## 🚀 快速开始

### 首次使用步骤

1. **安装应用**
   - 下载并安装静心自律应用
   - 打开应用后会自动进入主界面

2. **授予必要权限**
   - **应用使用统计权限**：用于监控应用使用情况
   - **通知权限**：显示后台运行状态和提醒
   - 在系统设置中授予"查看应用使用情况"权限

3. **设置时间段规则**
   - 点击首页"时段设置"按钮
   - 添加您的自由时段（如午休12:00-13:00）
   - 未设置规则的时段默认为限制时段

4. **创建时间模块**
   - 在设置页面点击"+"创建时间模块
   - 选择工作模式（额度模式或短时循环模式）
   - 配置使用时长和冷冻时长

5. **分配应用**
   - 编辑时间模块，添加需要管理的应用
   - 为不同应用选择合适的限制策略

### 2️⃣ 时间段规则管理

#### 什么是时间段规则？
时间段规则定义了一天中的**自由时段**，在这些时段内可以不受限制地使用所有应用。

#### 添加时间段规则

1. 进入"设置"页面
2. 点击时间段规则区域的"+"按钮
3. 配置以下参数：

**时间设置**
- **开始时间**：自由时段的起始时刻
- **结束时间**：自由时段的结束时刻
- 支持跨天设置（如22:00至次日02:00）

**快捷预设**
- **午休**：12:00-13:00
- **晚间**：18:00-22:00
- **全天**：00:00-23:59

**日期选择**
- 选择规则生效的星期几（周一至周日）
- 可多选（如仅工作日、仅周末等）
- 日期按钮分两行显示：第一行为周一至周五，第二行为周六和周日
- 点击对应的日期按钮即可选中或取消

4. 点击"确定"保存规则

#### 管理规则
- **启用/禁用**：使用开关临时启用或禁用某条规则
- **删除**：点击垃圾桶图标删除不需要的规则
- **多条规则**：可创建多条规则，任意一条匹配即为自由时段

#### 使用建议
- 为自己预留合理的放松时间
- 工作/学习时段设置为限制时段
- 根据个人作息定制个性化规则

---

### 3️⃣ 时间模块（TimeProfile）

#### 什么是时间模块？
时间模块是可复用的**应用限制配置模板**，包含工作模式、使用额度和冷冻时长等设置。

#### 两种工作模式

##### 📊 额度模式 (Quota Mode)
**适用场景**：希望严格控制每日总使用时长的用户

**工作机制**：
- 在限制时段内，每个应用每天有固定的使用额度
- 例如：设置30分钟额度，今天使用了30分钟后，该应用将被冷冻
- 次日零点额度自动重置
- 适合长期控制特定应用的使用频率

**配置项**：
- **短时使用时长**：每日可用额度（0-120分钟）
- **冷冻时长**：超限后的等待时间（5-30分钟）

##### 🔄 短时循环模式 (Cycle Mode)
**适用场景**：希望采用"番茄钟"式间歇使用的用户

**工作机制**：
- 允许短时间使用应用（如15分钟）
- 使用时间耗尽后进入冷冻期（如5分钟）
- 冷冻结束后额度自动恢复，可继续使用
- 形成"使用→冷冻→恢复→使用"的循环
- 适合需要频繁但短暂查看的应用（如消息类应用）

**配置项**：
- **短时使用时长**：单次可使用时长（0-120分钟）
- **冷冻时长**：每次循环的等待时间（5-30分钟）

#### 创建时间模块

1. 进入"设置"页面
2. 在"时间模块"区域点击"+"按钮
3. 填写配置信息：

**基本信息**
- **模块名称**：便于识别的名称（如"轻度限制"、"游戏专用"、"社交应用"）

**工作模式**
- 选择"额度模式"或"短时循环模式"

**时间设置**
- **额度时长**：根据模式不同，代表每日额度或单次时长
- **冷冻时长**：超限后的等待时间

4. 点击"保存"创建模块

#### 管理时间模块
- **编辑**：点击模块卡片修改配置
- **删除**：非默认模块可删除（默认模块不可删除）
- **默认模块**：系统提供一个默认模块，用于快速开始

### 4️⃣ 应用管理

#### 分配应用到时间模块

1. 在设置页面点击某个时间模块卡片
2. 进入模块编辑对话框
3. 在"管理的应用"区域点击"添加应用"
4. 从列表中选择要管理的应用
5. 应用将立即按照该模块的规则进行管理

#### 移除应用管理
- 在模块编辑对话框中，点击应用旁边的"×"按钮
- 该应用将不再受任何时间限制

#### 切换应用的模块
- 先从原模块中移除应用
- 再添加到新的模块中

#### 未管理的应用
- 未被分配到任何模块的应用不受限制
- 可在自由时段和限制时段自由使用

---

### 5️⃣ 冷冻机制

#### 什么是冷冻？
当应用在限制时段内超出设定的使用额度时，会触发**冷冻机制**：
- 全屏显示等待界面
- 禁止使用该应用
- 倒计时显示剩余等待时间
- 等待结束后自动解除限制

#### 冷冻期间
- **无法绕过**：禁用返回键，防止轻易退出
- **屏幕常亮**：保持显示以便查看倒计时
- **通知栏提示**：显示"冷冻中 XX分钟"
- **工作原理**：应用通过显示自己的全屏界面覆盖在其他应用之上，这是一种软性限制
- **注意**：用户仍可通过多任务切换或Home键离开冷冻界面，需要自律配合

#### 冷冻结束
- 倒计时归零后自动解除
- 可正常使用应用
- 短时循环模式下额度会重置

#### 应对建议
冷冻是**反思的机会，而非惩罚**：
- 深呼吸，放松身心
- 思考刚才的使用是否必要
- 起身活动，眺望远方
- 回顾自己的目标和计划

## 💡 使用技巧与最佳实践

### 🎯 新手入门建议

1. **循序渐进**
   - 第一周：仅设置自由时段，暂不限制应用
   - 第二周：为1-2个最耗时的应用设置温和限制
   - 第三周：逐步扩大管理范围

2. **合理设置额度**
   - 不要一开始就设置过严的限制
   - 观察一周的平均使用时长
   - 在此基础上减少20-30%作为初始额度

3. **预留缓冲时间**
   - 自由时段不要太少
   - 给自己留出放松和娱乐的空间
   - 避免产生逆反心理

### 📱 不同类型应用的配置建议

#### 社交媒体类（微信、微博、抖音等）
- **推荐模式**：短时循环模式
- **建议配置**：15分钟使用 + 5分钟冷冻
- **理由**：这类应用需要频繁查看，但不宜长时间沉浸

#### 游戏类
- **推荐模式**：额度模式
- **建议配置**：30-60分钟每日额度 + 15分钟冷冻
- **理由**：游戏容易上瘾，需要严格控制总时长

#### 学习工具类（词典、笔记等）
- **推荐模式**：不管理或使用宽松额度
- **建议配置**：60-120分钟额度
- **理由**：这些是生产力工具，不应过度限制

#### 工作通讯类（邮件、办公软件）
- **推荐模式**：不管理
- **理由**：工作需求，应保持可用性

```
- 可使用番茄钟配合短时循环模式
```

### 🔄 迭代优化流程

1. **第一周：观察期**
   - 设置宽松的自由时段
   - 暂时不限制应用或设置很宽松的额度
   - 重点关注统计数据，了解自己的使用模式

2. **第二周：调整期**
   - 根据数据找出最需要控制的应用
   - 为这些应用设置适中的额度
   - 微调自由时段

3. **第三周及以后：优化期**
   - 评估限制效果
   - 如果感觉太严，适当放宽
   - 如果感觉无效，适当收紧
   - 找到适合自己的平衡点

### ⚠️ 常见问题处理

#### Q1: 应用没有被限制？
**检查清单**：
- [ ] 该应用是否已分配到某个时间模块？
- [ ] 当前是否处于自由时段？
- [ ] 时间段规则是否正确设置？
- [ ] 监控服务是否在运行（通知栏有提示）？
- [ ] 是否授予了应用使用统计权限？

#### Q2: 冷冻界面可以绕过吗？
- 设计上已禁用返回键
- 部分系统可能允许通过多任务切换离开
- 建议培养自律意识，主动遵守规则
- 记住：绕过限制只会损害自己的目标

#### Q3: 应用如何“冻结”其他应用？需要特殊权限吗？
**答**：应用并不需要真正冻结或阻止其他应用，而是通过以下方式实现：
- **工作原理**：当检测到应用超限时，启动自己的全屏冷冻界面覆盖在当前应用之上
- **所需权限**：只需要 `PACKAGE_USAGE_STATS` 权限来监控哪个应用在前台
- **无需特殊权限**：显示自己的 Activity 是 Android 系统的正常功能，不需要系统级权限
- **软性限制**：这是一种基于自律的提醒机制，用户可以通过多任务或Home键离开
- **设计初衷**：给你一个停下来、深呼吸的机会，而非强制阻止

#### Q4: 如何临时关闭所有限制？
**方法一**：将所有时间段规则设置为自由时段
**方法二**：暂时禁用所有时间模块中的应用
**方法三**：在系统设置中停用监控服务（不推荐）

#### Q5: 统计数据不准确？
- 确保授予了正确的权限
- 某些系统可能需要额外设置
- 重启应用或设备后重试
- 检查是否有多个用户配置文件

#### Q6: 耗电量大吗？
- 应用经过优化，耗电量很低
- 每2秒检查一次，使用高效算法
- 实际测试显示日耗电<1%
- 如发现异常耗电，请反馈问题

## 🔐 隐私与安全

### 数据收集
- **本地存储**：所有数据仅存储在您的设备上
- **无网络请求**：应用不会上传任何数据到服务器
- **无广告追踪**：不包含任何第三方追踪代码

### 权限说明

#### 必需权限
- **PACKAGE_USAGE_STATS**：读取应用使用统计
  - 用途：监控哪个应用在前台运行及运行时长
  - 必要性：核心功能，必须授予

- **POST_NOTIFICATIONS**：发送通知
  - 用途：显示后台运行状态和冷冻提醒
  - 必要性：保证应用正常运行

- **FOREGROUND_SERVICE**：前台服务
  - 用途：持续监控应用使用
  - 必要性：确保后台稳定运行

- **RECEIVE_BOOT_COMPLETED**：接收开机广播
  - 用途：设备重启后自动启动服务
  - 必要性：保证全天候保护

- **QUERY_ALL_PACKAGES**：查询所有应用
  - 用途：获取已安装应用列表供用户选择
  - 必要性：应用管理功能需要

### 数据安全
- 数据库文件加密存储
- 不使用外部存储
- 卸载应用后数据自动清除

## 🆘 故障排除

### 应用无法启动监控服务

**症状**：通知栏没有显示监控运行提示

**解决方案**：
1. 检查是否授予了所有必要权限
2. 进入系统设置 → 应用 → 静心自律 → 权限，确保"应用使用情况"已授权
3. 重启应用
4. 如仍无效，重启设备

### 冷冻界面无法正常显示

**症状**：超限时没有弹出冷冻界面

**解决方案**：
1. 检查是否禁用了后台活动限制
2. 在电池优化设置中将应用设为"不优化"
3. 锁定应用在多任务列表中（防止被清理）
4. 检查是否有其他全屏应用干扰

### 统计数据为空或不更新

**症状**：今日使用时长始终为0或不变化

**解决方案**：
1. 确认已授予"应用使用情况"权限
2. 尝试手动触发：打开一个被管理的应用，使用几分钟
3. 检查该应用是否真的被分配到时间模块
4. 查看Logcat日志排查错误

### 应用被系统杀死

**症状**：一段时间后监控停止

**解决方案**：
1. 在电池设置中关闭对该应用的省电优化
2. 在多任务界面锁定应用
3. 关闭"自动清理后台"功能
4. 不同品牌手机设置路径不同，请搜索"[手机品牌] 后台保活设置"

### 权限授予后仍提示无权限

**症状**：已授予权限但应用仍提示需要权限

**解决方案**：
1. 完全关闭应用后重新打开
2. 进入系统设置撤销权限后重新授予
3. 重启设备
4. 检查是否有多个用户账户，需在当前账户下授权
